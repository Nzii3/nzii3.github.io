Discord's Big Announcement

A little over a year ago, Discord announced the release of slash commands. A little while after the release of slash commands, Discord released an announcement for Discord Developers that Discord will be removing the message.content from verified bots unless they apply for it. No Bot Developer liked this decision, no one. However, in that announcement, Discord said they were going to make more UI features to replace the need for fetching message content. This means Discord Auto Moderation and Discord Anti-Spam bots would no longer be able to use their main functionality without applying for this intent. From what I've heard, they are very strict to whom they give the intent to. I honestly have no clue why they removed it, maybe it uses too much resources? If so, then they probably would have removed it from non-verified bots too, removing it from the API

Along with this announcement, Danny (discord.py library developer) decided to archive the discord.py library as public, so other library developers could have a chance to make an alternative using his code. He did this because he didn't agree with the path Discord was taking for "the long road." No one really agreed with Discord either, but there's nothing we can do about it and Discord have made up their mind.

An amazing library rose from Danny making the discord.py source code public: Pycord. This truly is an amazing library and is updated frequently when Discord releases new features. In my opinion, it's much better than discord.py because there are more contributors and maintainers, meaning more updates and bug fixes than discord.py. However, Danny gets all the credit as deserved for making the base code of Pycord, discord.py v2. I have been using Pycord for all my bots since the week it was released. Writing code for Discord bots in Python was saved...
